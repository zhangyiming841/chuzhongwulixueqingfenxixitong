## 1. 项目当前定位

这是一个用于展示“初中物理学情分析”的前端演示版项目，目标是：

- 保留原有大屏展示风格
- 不依赖真实 API 也能运行
- 用本地规则引擎模拟 AI 诊断流程
- 支持课堂对话录入、知识点识别、误解判断、苏格拉底式追问、学情报告和人工评估
- 适合本地运行、演示汇报、部署到静态服务器展示

当前版本属于：

- 可运行演示版
- 本地 mock 诊断版
- 非正式生产版

## 2. 当前已实现的核心功能

### 2.1 对话录入

已支持以下录入方式：

- 手动输入老师与学生的对话内容
- 文本文件上传
- 浏览器语音录入
- 一键切换演示示例

其中“演示示例”支持快速切换不同知识点场景，方便现场展示不同诊断结果。

### 2.2 智能诊断

当前诊断逻辑运行在前端本地，不依赖真实大模型接口。  
系统会根据输入内容进行：

- 知识点识别
- 章节归属判断
- 掌握层级判断
- 认知阻滞点识别
- 苏格拉底式引导问题生成
- 教师追问建议生成
- 学情总结生成

### 2.3 苏格拉底式提问

右侧第三栏已从“默认直接显示学情报告”改为：

- 先展示 2 到 3 个苏格拉底式引导问题
- 问题围绕当前识别到的知识点展开
- 更符合“老师继续追问学生”的教学节奏

### 2.4 学情报告

学情报告已改成二段式流程：

- 先完成诊断
- 先展示苏格拉底式问题
- 延迟 3 秒后弹出“是否查看学情报告”的确认框
- 用户确认后再打开完整学情报告弹窗

### 2.5 知识库管理

设置按钮已接入知识库管理弹层，当前支持：

- 查看系统当前知识覆盖范围
- 查看章节数、知识点数、条目总数
- 以“知识点聚合视图”查看概念定义、常见误解、引导策略
- 手动新增知识条目

### 2.6 人工评估

已提供人工评估弹层，支持：

- 对当前诊断结果进行老师复核
- 概念判断准确度评分
- 阻滞点识别评分
- 引导问题有效性评分
- 输出格式稳定性评分
- 自动计算综合得分
- 自动给出评估建议
- 记录老师备注

### 2.7 历史与本地存储

当前所有演示数据都存储在浏览器本地，包括：

- 诊断记录
- 人工评估记录
- 本地知识库扩展内容

这意味着：

- 不需要后端就能演示
- 刷新页面后数据通常仍在
- 更换浏览器或清空本地存储后数据会丢失

## 3. 当前知识点覆盖范围

本地 mock 诊断已扩展到初中物理主干知识点，当前覆盖包括：

### 力学

- 惯性
- 平衡力与相互作用力
- 摩擦力
- 密度
- 压强
- 浮力
- 功和功率
- 机械能及其转化
- 简单机械

### 热学

- 物态变化
- 内能与热传递

### 光学

- 光的反射
- 光的折射
- 凸透镜成像

### 电学与磁学

- 电流与电压
- 电阻与欧姆定律
- 串并联电路
- 电功率与焦耳热
- 磁现象与电磁感应

### 兜底逻辑

还加入了“综合判断”兜底：

- 当输入太模糊
- 当问题跨多个知识点
- 当无法稳定命中单一概念

系统不会硬判错，而会输出更保守的综合判断结果。

## 4. 当前页面设计结构

项目主界面为三栏结构：

### 左栏：对话内容录入

包含：

- 学生信息
- 会话编号
- 演示示例切换
- 文本输入区
- 文本上传
- 语音录入
- 开始分析按钮
- 清空按钮

### 中栏：AI 智能解析

已从原来“中间大面积空白 + 图标”的状态改为：

- Live Status 状态卡
- 当前焦点卡
- 当前判断卡
- 三步分析过程卡

三步分析包括：

- 识别知识点
- 识别阻滞点
- 生成引导建议

并且已经加入：

- 分析时逐步点亮
- 完成后显示已完成状态
- 更接近真实 AI 处理流程的视觉节奏

### 右栏：苏格拉底式提问

包含：

- 当前识别知识点
- 2 到 3 个引导问题
- 提问思路说明
- 学情报告入口

## 5. 当前演示增强设计

为了提高“可演示性”和“可发现性”，已经补了以下设计：

- 首次进入时的轻提示气泡
- 更显眼的知识库管理按钮
- 更显眼的人工评估按钮
- 可直接切换的演示示例
- 学情报告延迟弹出逻辑
- 更完整的 AI 分析中栏

这些设计的目标是让第一次看到页面的人也能快速理解：

- 系统能做什么
- 从哪里进入核心功能
- AI 正在做什么
- 老师如何参与复核

## 7. 当前不依赖 API 的原因

当前版本不接 API 也能演示，原因是：

- 知识点识别逻辑已经本地化
- 学情分析输出已经本地化
- 学情报告和追问生成已经本地化
- 知识库与人工评估也都在前端本地完成

因此当前版本具备：

- 可本地运行
- 可打包构建
- 可部署静态页面
- 可用于汇报和演示

## 9. 当前已知限制

虽然已经很适合演示，但当前仍有这些限制：

- 不是真实大模型推理
- 不是多人协作系统
- 数据不跨设备同步
- 本地存储可能因清缓存丢失
- 对极复杂、跨章节、极口语化问题的判断仍是“近似真实”，不是百分之百准确
- 部分历史代码中仍存在旧版残留结构，后续可以继续清理

## 10. 下一步建议

如果继续迭代，优先级建议如下：

- 统一清理剩余乱码文案
- 继续补更多课堂常见错误表达

- 按力学 / 光学 / 电学对示例分类
- 让知识库支持编辑已有条目
- 让学情报告更像正式报告版式

### 优先级 3

- 接后端 API
- 接数据库
- 支持多人共享
- 支持正式线上产品化
